# Профили участников марафона (Bridge / Bloom)

## Зачем это

Дальше развиваем **общение с каждым пользователем**: Bloom будет опираться на историю и на «личность» участника. Данные со временем **растут**, поэтому **не храним всё в одном общем текстовом файле** — у каждого человека своя папка.

## Правило именования папок

- Для каждого пользователя создаётся **отдельная директория**.
- Имя папки — по **имени и фамилии из данных Telegram** (если они есть), в читаемом виде; при необходимости можно добавить суффикс `_telegram_id`, чтобы не путать однофамильцев.
- Если имени/фамилии нет — используем `@username` или `user_<id>`.

## Файлы внутри папки (сейчас)

| Файл | Назначение |
|------|------------|
| `chat_history.txt` | История переписки с Bloom (серверные заметки, не дубли dialogy при желании) |
| `biography.txt` | **Краткая справка** — основа для промпта (держи сжато, чтобы не забивать контекст) |
| `biography_full.txt` | **Полная справка** (опционально) — дополнительный материал; в коде подмешивается фрагментом при запросах Максу «по базе» |

Временные материалы (скрины, выгрузки) можно класть в **`inbox/`** у корня проекта; после переноса сути в `biography.txt` / `biography_full.txt` — **файлы из inbox удали** (см. `inbox/README.md`).

Формат текстов потом можно сменить на БД; переменные окружения для **PostgreSQL** уже задаются в `.env` (`POSTGRES_*`).

## Приоритет разработки

Сначала доводим **личные профили и диалоги** (`bridge_participants`, `dialogy`). Поведение **в группах марафона** — отдельным этапом позже.

## Подключение к PostgreSQL из кода

Параметры читаются из окружения (см. `.env`):

- `POSTGRES_HOST`
- `POSTGRES_PORT` (по умолчанию 5432)
- `POSTGRES_USER`
- `POSTGRES_PASSWORD` (значения с `#`, `&` и т.п. — в **кавычках** в `.env`)
- `POSTGRES_DB`

Пример строки подключения (после загрузки `dotenv`):

```python
import os
url = (
    f"postgresql://{os.environ['POSTGRES_USER']}:{os.environ['POSTGRES_PASSWORD']}"
    f"@{os.environ['POSTGRES_HOST']}:{os.environ.get('POSTGRES_PORT', '5432')}/{os.environ['POSTGRES_DB']}"
)
```

Для URL лучше использовать кодирование пароля (`urllib.parse.quote_plus`), если пароль содержит спецсимволы.
