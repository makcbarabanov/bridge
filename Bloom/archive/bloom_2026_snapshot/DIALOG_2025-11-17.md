# Диалог с Bloom - 17 ноября 2025

## Проблемы с Caddy и деплоем bloom.html

### Вопросы пользователя:

2. Поиск способов перезапуска Caddy

### Решенные проблемы:

#### 1. Проверка кода bloom.html
- Пользователь показал HTML-код страницы марафона, подумал что это код bloom.html
- Выяснили: на сервере `/home/www/public/bloom.html` содержал страницу марафона, а не Bloom
- Локальный файл `index.html` содержит правильную страницу Bloom

#### 2. Проблема с ERR_QUIC_PROTOCOL_ERROR
- Ошибка возникала потому что браузер пытался открыть HTTPS, а Caddy был настроен только на HTTP
- **Решение**: Открывать по HTTP (без S): `http://marabot.tw1.ru`

#### 3. Запуск Caddy
- Caddy не был запущен на сервере
- Проверили Docker контейнеры - Caddy не в Docker
- Создали Caddyfile: `/etc/caddy/Caddyfile`
- Запустили Caddy вручную:
```bash
nohup caddy run --config /etc/caddy/Caddyfile > /var/log/caddy.log 2>&1 &
```

#### 4. Конфигурация Caddyfile
```caddyfile
{
    auto_https off
}

:80 {
    root * /home/www/public
    file_server
}
```

### Состояние сервера:
- Caddy запущен (PID: 1311072)
- Слушает порт 80
- Файлы находятся в `/home/www/public/`
