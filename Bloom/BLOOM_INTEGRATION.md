# Bloom в bridge_bot

## Канон

Все тексты личности и памяти читаются **только** из этого каталога (`Bloom/`), по умолчанию:

`Bloom/extracted/BLOOM 2026/`

Переопределение: переменная окружения **`BLOOM_HOME`** (абсолютный путь к папке `BLOOM 2026`).

## Файлы, которые использует бот

| Файл | Назначение |
|------|------------|
| `BLOOM_START_PROMPT.txt` | База для `systemInstruction` в Gemini |
| `README.md` | Начало файла + миссия (обрезка по длине) |
| `DIALOG_BLOOM_FORGE_letters-with-bloom.txt` | Поиск выдержек по ключевым словам из сообщения |

Новые блоки памяти можно класть в `Bloom/` и при необходимости расширить `bloom_context.py`.

## Группа марафона

В `.env` задай **`MARATHON_GROUP_IDS`** — id супергруппы (отрицательное число, напр. `-1001234567890`).

В группе Bloom отвечает, если:

- сообщение с **командой** (`/start`, `/help`, `/bloom`, …), или  
- есть **@username** бота, или  
- **ответ** (reply) на сообщение бота.

Команда **«ЗАПОМНИ»** и запись в `brain_dump.txt` — **только в личном чате** с Максом.

## Обновление памяти

После добавления больших файлов в канон можно вызвать в коде `bloom_context.invalidate_cache()` (пока не вызывается автоматически — перезапуск бота достаточно).

## Повторяющийся «ритуал» в ответах

В начале `systemInstruction` зашит блок **режим TELEGRAM**: без шаблона «Память восстановлена» и трёх маркеров на каждое сообщение. В `BLOOM_START_PROMPT.txt` формат «первого ответа» помечен как только для Cursor/Studio. После правок промпта перезапусти бота (кэш сбросится).
