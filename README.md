# Bridge — Telegram-бот Bloom («Мост»)

## Назначение

Рантайм: **aiogram** + опционально **Gemini** / **Groq**, лог диалогов, участники марафона, дайджест мечт из **PostgreSQL**.

## Структура репозитория

```
bridge/
├── bridge_bot.py          # Точка входа бота, хендлеры, AI, фоновые задачи
├── dream_db.py            # Запросы к БД (мечты) — только то, что нужно боту
├── bridge_participants.py # Участники, user_profiles, хвост chat_history в промпт
├── bridge                 # Скрипт: systemctl --user для bridge-bot.service
├── Bloom/                 # Канон Блума: bloom_context, распакованный архив проекта
├── book_assembly/       # Книга «14:45» — см. book_assembly/README.md (отдельный трек)
├── user_profiles/       # Папка на человека: biography.txt, chat_history.txt (лог в .gitignore)
├── inbox/               # Временные файлы (в git только README)
├── _py_/                # Утилиты и разовые скрипты — см. _py_/README.md
├── .env.example
└── venv/                # не в git
```

## Принципы

1. **Корень** — только то, что нужно для **запуска** бота и общих модулей. Новые фичи бота — рядом с `bridge_bot.py` или отдельным модулем с понятным именем (`dream_db.py`), без свалки скриптов.
2. **`_py_/`** — всё, что запускается руками (список моделей, тестовая отправка в Telegram). Разовое — удалить после использования.
3. **`book_assembly/`** — автономный контур «книги»; не смешивать с логикой Telegram.
4. **Секреты** — только в `.env` (в git не попадает). Пример — `.env.example`.

## Профили и лог диалога (`user_profiles/`)

- **Участники** из `bridge_participants.PARTICIPANTS` — в своей папке (поле `folder`), например `user_profiles/Макс/`.
- **Гости** без записи в карте — отдельная папка `user_profiles/<имя_из_Telegram>_<telegram_id>/` (если уже есть каталог `*_<id>` после миграции, используется он).
- **`chat_history.txt`** — единственный автолог всей переписки с ботом; хвост подмешивается в промпт. Файл в `.gitignore` (не коммитить с сервера).
- **`biography.txt`** — справка о человеке; при первом контакте создаётся пустым, если файла ещё нет (дальше заполняется вручную).
- При **`/start`** в личке бот вызывает `ensure_profile`: создаёт папку при необходимости и пустой `biography.txt`. Повторный «Старт» после удаления чата не создаёт новый профиль: пользователь с тем же `telegram_id` находится по карте или по суффиксу `_<id>` в имени папки.
- Разовая миграция из старой схемы `dialogy/` + `dialog_with_bloom.txt`: см. `_py_/migrate_dialogy_to_chat_history.py` (уже выполнена в репозитории разработки).

## Запуск

```bash
source venv/bin/activate
python bridge_bot.py
# или через systemd: bridge start | restart | status | logs
```
